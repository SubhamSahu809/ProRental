# Frontend File Structure & Architecture

## 📁 Directory Structure Overview

```
Frontend/
├── public/              # Static assets served directly
├── src/                 # Source code (all React components)
│   ├── assets/         # Images and other assets used in components
│   ├── components/     # Reusable React components
│   └── pages/          # Page-level components (routes)
├── dist/               # Build output (generated by Vite)
├── node_modules/       # Dependencies (installed packages)
└── Configuration files # Build tools and config
```

## 🎯 Core Files Explained

### **Entry Point Files**

#### 1. `index.html` (Root HTML Template)
```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>ProRental</title>
  </head>
  <body>
    <div id="root"></div>  ← React app mounts here
    <script type="module" src="/src/main.jsx"></script>  ← Entry point
  </body>
</html>
```
- **Purpose**: The HTML shell that contains the React app
- **Key Element**: `<div id="root">` - where React renders everything
- **Script**: Loads `main.jsx` as the entry point

#### 2. `src/main.jsx` (Application Entry Point)
```jsx
import { createRoot } from 'react-dom/client'
import App from './App.jsx'
import { BrowserRouter } from 'react-router-dom'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>
)
```
- **Purpose**: Initializes React app and renders it to the DOM
- **What it does**:
  1. Creates a React root at `#root` element
  2. Wraps app with `BrowserRouter` for routing
  3. Renders the main `App` component
- **Execution Flow**: `index.html` → `main.jsx` → `App.jsx` → Your Pages

#### 3. `src/App.jsx` (Main Application Component)
```jsx
function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/add-property" element={<AddPropertyPage />} />
      <Route path="/login" element={<LoginPage />} />
      <Route path="/property/:id" element={<Property />} />
      <Route path="/my-properties" element={<MyProperties />} />
    </Routes>
  );
}
```
- **Purpose**: Defines all application routes
- **Routing**: Uses React Router to map URLs to page components
- **Routes**:
  - `/` → Home page
  - `/add-property` → Add Property page
  - `/login` → Login page
  - `/property/:id` → Individual property detail page
  - `/my-properties` → User's property list

## 📦 Components vs Pages

### **Components** (`src/components/`)
Reusable UI building blocks used across multiple pages:

1. **Navbar.jsx** - Navigation bar (appears on all pages)
2. **Footer.jsx** - Footer section
3. **Hero.jsx** - Hero/banner section for homepage
4. **Property.jsx** - Individual property display component
5. **AddProperty.jsx** - Form component for adding properties
6. **LoginModal.jsx** - Login modal dialog
7. **SignupModal.jsx** - Signup modal dialog
8. **Services.jsx** - Services section component

**Example Component Usage**:
```jsx
// In Home.jsx
import Navbar from "../components/Navbar";
import Footer from "../components/Footer";

function Home() {
  return (
    <div>
      <Navbar />    {/* Reusable component */}
      <Hero />      {/* Reusable component */}
      <Footer />    {/* Reusable component */}
    </div>
  );
}
```

### **Pages** (`src/pages/`)
Full-page components that correspond to routes:

1. **Home.jsx** - Landing page (combines Navbar, Hero, Properties, Contact, Footer)
2. **AddProperty.jsx** - Page for adding new properties
3. **Login.jsx** - Login page (renders LoginModal)
4. **MyProperties.jsx** - User's property management page
5. **Proporties.jsx** - Property listing display
6. **Contact.jsx** - Contact form section

**Example Page Structure**:
```jsx
// Home.jsx - Composes multiple components
function Home() {
  return (
    <div>
      <Navbar />        {/* Component */}
      <Hero />          {/* Component */}
      <Proporties />    {/* Page-like component */}
      <Contact />       {/* Page-like component */}
      <Footer />        {/* Component */}
    </div>
  );
}
```

## ⚙️ Configuration Files

### **1. `package.json`** (Project Dependencies & Scripts)
```json
{
  "scripts": {
    "dev": "vite",           // Start development server
    "build": "vite build",   // Build for production
    "preview": "vite preview" // Preview production build
  },
  "dependencies": {
    "react": "^19.1.1",           // React library
    "react-dom": "^19.1.1",       // React DOM rendering
    "react-router-dom": "^7.7.1"  // Client-side routing
  }
}
```
- **Purpose**: Defines project dependencies and npm scripts
- **Key Dependencies**:
  - `react` & `react-dom`: Core React framework
  - `react-router-dom`: Client-side routing
  - `lucide-react`: Icon library

### **2. `vite.config.js`** (Build Tool Configuration)
```js
export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,      // Dev server port
    host: true       // Accessible from network
  }
})
```
- **Purpose**: Configures Vite build tool
- **Vite**: Fast build tool and dev server for React
- **What it does**:
  - Enables React plugin for JSX/TSX support
  - Sets dev server to run on port 5173
  - Allows network access (`host: true`)

### **3. `tailwind.config.js`** (CSS Framework Configuration)
```js
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}"  // Scan these files for Tailwind classes
  ]
}
```
- **Purpose**: Configures Tailwind CSS utility framework
- **Tailwind CSS**: Utility-first CSS framework
- **How it works**: Scans your JSX files for Tailwind classes and generates CSS

### **4. `postcss.config.js`** (CSS Processing)
- **Purpose**: Processes CSS (used by Tailwind)
- **PostCSS**: Transforms CSS with JavaScript plugins

### **5. `eslint.config.js`** (Code Quality)
- **Purpose**: Linting rules for code quality
- **ESLint**: Catches errors and enforces coding standards

## 🔄 Application Flow

### **Development Workflow**

1. **Start Dev Server**: `npm run dev`
   - Vite starts development server on `http://localhost:5173`
   - Hot Module Replacement (HMR) - changes reflect instantly

2. **User Visits URL**: `http://localhost:5173/`
   - Browser loads `index.html`
   - HTML loads `main.jsx` script

3. **React Initialization**:
   ```
   index.html → main.jsx → BrowserRouter → App.jsx → Routes → Home.jsx
   ```

4. **Component Rendering**:
   ```
   Home.jsx renders:
   ├── Navbar (with LoginModal, SignupModal)
   ├── Hero section
   ├── Proporties (fetches and displays properties)
   ├── Contact section
   └── Footer
   ```

5. **Navigation**:
   - User clicks link → React Router changes URL
   - Router matches URL to route in `App.jsx`
   - Corresponding page component renders

### **Production Build**

1. **Build Command**: `npm run build`
   - Vite bundles all code into optimized files
   - Outputs to `dist/` directory:
     - `dist/index.html` - Main HTML
     - `dist/assets/index-[hash].js` - Bundled JavaScript
     - `dist/assets/index-[hash].css` - Bundled CSS
     - `dist/assets/` - Images and other assets

2. **Deployment**:
   - `dist/` folder is deployed to hosting (Vercel, Netlify, etc.)
   - Server serves the static files

## 🗂️ File Organization Patterns

### **Import Paths**
```jsx
// Relative imports (same directory)
import Component from "./Component"

// Parent directory imports
import Component from "../components/Component"

// Asset imports
import image from "../assets/image.jpg"
```

### **Naming Conventions**
- **Components**: PascalCase (e.g., `Navbar.jsx`, `LoginModal.jsx`)
- **Pages**: PascalCase (e.g., `Home.jsx`, `MyProperties.jsx`)
- **Files**: Match component name exactly (case-sensitive for Linux)

### **Component Structure**
```jsx
// 1. Imports
import React, { useState } from "react"
import Component from "./Component"

// 2. Component Definition
const MyComponent = () => {
  // 3. State & Hooks
  const [state, setState] = useState(null)
  
  // 4. Functions
  const handleClick = () => { ... }
  
  // 5. Render
  return <div>...</div>
}

// 6. Export
export default MyComponent
```

## 🔌 API Integration

### **Backend Communication**
Components fetch data from backend API:
```jsx
const response = await fetch("http://localhost:8080/api/listings", {
  credentials: "include"  // Includes cookies for authentication
})
const data = await response.json()
```

### **Common API Endpoints Used**:
- `GET /api/listings` - Fetch all properties
- `GET /api/listings/:id` - Fetch single property
- `POST /api/listings` - Create new property
- `DELETE /api/listings/:id` - Delete property
- `GET /users/me` - Get current user
- `POST /users/login` - User login
- `POST /users/signup` - User registration

## 📝 Best Practices in This Project

1. **Component Separation**: Logic separated into reusable components
2. **Route-Based Pages**: Each route has its own page component
3. **Responsive Design**: Tailwind CSS classes for mobile-first design
4. **State Management**: React hooks (`useState`, `useEffect`) for local state
5. **Authentication**: Cookie-based auth with `credentials: "include"`
